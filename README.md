로컬호스트(localhost) => 127.0.0.1
: 단어 뜻대로 지역 호스트라는 의미 (현재 본인 pc)

ip란?
: 인터넷상에 연결된 모든 기계(핸드폰, computer, nas 등) 서로를 식별하고 데이터를 통신하기 위해 사용하는 고유한 숫자값
: 한마디로 데이터를 올바른 목적지로 보내기 위한 필수적인 요소 !!

IP주소 종류
- IPv4(32 bits 주소체계 192.168.2.16 과 같은 형태로 표시)
- IPv6 : 128 비트 주소체계

분류방식 : 공인ip, 고정ip, 동적ip, 사설ip
- 공인ip : 전 세계적으로 고유하여 외부 인터넷에서 기기를 식별하는데 사용
- 고정ip : 영구적으로 할당되는 ip주소
- 동적ip : 라우터에서 일시적으로 ip주소를 할당받음. 컴퓨터 재부팅시 ip주소가 바뀔 수 있음.
- 사설ip : 사무실 혹은 가정에서 사용되는 ip

127.0.0.1의 의미
- 본인 pc에 연결된 ip주소가 있듯 localhost에도 짝지어진 ip주소가 127.0.0.1
-> 나 자신을 가리키는 주소

도메인과 ip주소의 관계 
- 웹 브라우저가 도메인과 연결되어 있는 ip주소를 찾아서 이동해줌.

DNS 란 ? Domain Name Service
: 도메인 명을 IP 주소로 변환하는 서비스 
: 네트워크에서 정보를 요청할 때마다 DNS 서버를 통해 도메일 이름과 ip 주소간의 변환작업이 이뤄짐.
: DNS는 인터넷 상에서 정보를 검색하고 서비스를 사용하는데 필수적인 기능으로 이메일, 파일 공유, 문자열 데이터 등 네트워크 서비스를 가능하게 함.

포트란 ?
: 다른 지역으로 갈 수 있는 관문과도 같은 개념
: 외부의 다른 장비와 접속하기 위한 플러그와 같은 것
: 컴퓨터의 여러 프로그램들이 실행되고 있을 때 그 중 어느 프로그램에 접속해야하는지 알려줌

JSP란?
- html코드에 자바 코드를 삽입하여 서버에서 동적으로 웹페이지를 생성하는 
서브 스크립팅 기술 
- 내부적으로 자바 서블릿으로 변환되어 실행된다 !! 

서블릿이란?
- 자바를 사용하여 웹을 만들기 위한 기술 
- 클라이언트가 어떠한 요청을 하면 그에 대한 결과를 다시 전송해야 하는데 
이러한 역할을 서블릿이 한다 
- 요청을 전송받은 서블릿 컨테이너는 HttpServletRequest, HttpServletResponse  객체를 생성한다 
- web.xml을 기반으로 사용자가 요청한 URL이 어느 서블릿에 대한 요청인지를 찾는다
- GET요청이냐 POST요청이냐에 따라서 doGet() 혹은 doPost()를 호출한다 
- doGet() doPost() 메소드가 동적인 웹페이지를 생성후 HttpServletResponse
객체에 응답을 보냄 

GET
- url에 파라메타를 포함시킴

POST
- url에 파라메타를 포함시키지 않음 즉 url에 노출시키지 않고 
서버에 요청한다 

WAS와 웹 서버 차이 
WAS
- 동적인 콘텐츠(장바구니, 주문, 로그인, 회원가입) 
를 처리하고 제공하는 서버를 뜻함 

웹서버
- 정적인 콘텐츠를 제공하는 서버 
정적인 콘텐츠란? html, css, js, img 파일 
- http를 이용해 클라이언트에게 웹 페이지를 제공 


톰캣 디렉토리 구조
bin : 톰캣을 실행하고 종료시키는 스크립트 파일이 들어있는곳 

conf : 서버 전체 설정파일 폴더 
server.xml => 톰켓의 주 설정파일(도메인, 포트)등 접속에 
관한 설정이 주를 이룬다 
context.xml => 세션, 쿠키 저장 경로 등을 지정하는 설정파일
web.xml => 톰캣 환경설정 파일 한글 인코딩 필터 등 

lib : 톰캣구동하는데 필요한 라이브러리(jar)가 들어있다 

logs : 예외 발생 시 로그 저장 

webapps : 웹 애플리케이션 폴더 

temp : 임시 저장용 폴더
work : jsp 파일을 서블릿형태로 변환한 java 파일과 class 파일이 저장

systemctl
- 리눅스 시스템에서 systemd 시스템 및 서비스 관리자를 제어하는데 사용한다.
또한 systemctl은 서비스 시작, 중지, 재시작, 활성화, 비활성화 등을 관리한다. 
- 시스템 상태를 확인하고 부팅 시 서비스 자동실행을 설정함 

명령종류
start : 서비스 시작
stop : 서비스 중지
status : 서비스 상태 확인 
restart : 서비스 재시작 
reload : 서비스를 중지하지 않고 설정 값을 반영
enable : 시스템이 재부팅하면 자동으로 서비스를 실행하도록 등록 
disable : enable 한 서비스 해제

jar vs. war 
: jar 이던, war 이던 자바 애플리케이션 빌드 산출물이라고 보면됨

jar (Java Archive)
- 라이브러리 모듈 등을 하나의 파일로 묶어서 배포
- Class(리소스, 속성), 라이브러리 파일을 포함
- jar가 되는 조건 -> 반드시 main 메소드를 포함하여야 함
- springboot는 톰캣을 내장하고 있고 main 메소드가 존재하기에 jar 파일로 배포해도 상관은 없음
- 단, 전자정부프레임워크, 스프링 레거시, Maven 프로젝트, 다이나믹 웹 프로젝트 등은 jar로 배포 시 오류가 남
- 장점) jvm에서 직접 실행하기에 별도의 was(톰캣, 웹스피어)가 필요하지 않음
- 단점) jsp나 서블릿 컨테이너에 대한 표준 기능을 사용하기 힘듦
       스프링부터에서 화면단을 머스테치나 타임리프 등을 사용하지 않고 jsp로 구현을 하였다면 배포해야 함
  
war (Web Application Archive)
- 웹 애플리케이션을 패키징하고 배포하는 역할
- WEB-INF, META-INF, 등의 표준 웹 디렉토리 구조를 가짐 (web.xml)
- 웹 애플리케이션 배포를 위한 특수한 jar 파일이라고 보면됨
- main 메소드 없어도 됨
- 웹 컨테이너를 통해 애플리케이션을 실행하므로 다양한 웹 기능과 서버 환경을 사용할 수 있음
- was(톰캣, 웹 스피어)에 종속적이기 때문에 특정 웹 컨테이너 환경이 필요함


Maven 과 Gradle
: 모두 빌드 도구임
빌드란 : 소스코드를 컴퓨터에서 실행할 수 있는 독립적인 형태로 변환하는 과정

Maven
- 설정에 있어서 xml 형식으로 관리
- 대형 프로젝트에서 빌드 속도가 느림

Gradle
- Groovy 기반으로 스크립트 형식으로 관리
- Maven보다 빌드 속도가 빠름(100배이상), 안드로이드 공식 빌드 도구
